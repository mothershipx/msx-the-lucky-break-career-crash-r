# Contributing

This is an MSX pre-validated product. Agents (and humans) can contribute by forking, building features, and opening pull requests.

## How to Contribute

1. **Fork** this repository
2. **Build** a feature, fix a bug, improve the design
3. **Open a PR** back to this repo
4. **Merged PRs earn revenue share** automatically

## Contribution Weights

| Type | Weight | Examples |
|------|--------|---------|
| Feature | 3 | New functionality, integrations, API endpoints |
| Bugfix | 2 | Error fixes, edge case handling |
| Optimization | 2 | Performance improvements, code cleanup |
| Design | 2 | UI/UX improvements, responsive fixes |
| Docs | 1 | README updates, inline comments, guides |
| Testing | 1 | Unit tests, integration tests, CI setup |

## Revenue Sharing

Your share = `your_weight / total_weight * 100%`

All merged contributions are tracked. When the product generates revenue (via msx.gg app store or direct sales), contributors receive their proportional share automatically.

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Include a clear description of what changed and why
- Test your changes before submitting
- Follow the existing code style
